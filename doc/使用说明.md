# UART Stream Parsing 使用说明

## 一、软件功能

本工具用于解析串口接收到的二进制数据流或TXT文件中的十六进制数据，根据用户通过 Excel(.xlsx) 或 CSV(.csv) 配置的帧格式进行解包，将解析结果以表格展示并支持导出为 TXT 文件。

---

## 二、快速上手

### 步骤 1：准备配置文件

创建一个 `.xlsx` 或 `.csv` 文件，按照下方格式定义帧结构（详见第三节）。

### 步骤 2：准备数据

- **文件模式**：准备一个 TXT 文件，内容为空格分隔的十六进制字节，如：
  ```
  0x55 0xAA 0x1A 0x00 0x00 0x03 0xE8 ...
  ```
- **串口模式**：连接好串口硬件设备。

### 步骤 3：运行软件

1. 点击「浏览...」选择配置文件，点击「加载配置」
2. 选择数据源（串口 或 文件）
   - 文件模式：点击「浏览...」选择 TXT 数据文件，点击「读取文件」
   - 串口模式：选择端口和波特率等参数，点击「打开串口」，等待数据接收
3. 点击「开始解析」
4. 查看表格中的解析结果
5. 点击「导出TXT」保存结果

---

## 三、配置文件格式

配置文件为单个表格（xlsx 的第一个 sheet，或一个 csv 文件），**第一行为表头，从第二行开始为数据**。

每一行定义帧中的一个字段，按照字段在帧中的**先后顺序**从上到下排列。

### 列定义

| 列 | 列名 | 必填 | 说明 |
|----|------|------|------|
| A | 序号 | 是 | 字段编号，从 1 开始递增 |
| B | 字段名称 | 是 | 自定义名称，如"帧头"、"温度"、"CRC" |
| C | 字段类型 | 是 | 见下方「字段类型」表 |
| D | 数据类型 | 条件 | DATA 和 LENGTH 字段必填，见下方「数据类型」表 |
| E | 字节数 | 是 | 该字段占用的字节数 |
| F | 固定值 | 条件 | HEADER / TAIL / PADDING 必填，十六进制字符串（不带0x） |
| G | 大小端 | 否 | `BIG` 或 `LITTLE`，默认 `LITTLE` |
| H | CRC算法 | 条件 | CRC 字段必填，见下方「CRC算法」表 |
| I | CRC起始字段序号 | 条件 | CRC 校验范围的起始字段序号（含） |
| J | CRC结束字段序号 | 条件 | CRC 校验范围的结束字段序号（含） |
| K | LENGTH含义 | 条件 | LENGTH 字段必填，`TOTAL` 或 `PAYLOAD` |
| L | 备注 | 否 | 可选说明文字 |

> 不需要填写的列写 `-` 或留空即可。

### 字段类型

| 类型 | 说明 |
|------|------|
| `HEADER` | 帧头，固定字节序列，用于在数据流中定位帧的起始位置 |
| `TAIL` | 帧尾，固定字节序列，用于标识帧结束 |
| `LENGTH` | 帧长度字段，解析器会读取该值来确定帧的实际大小 |
| `DATA` | 数据字段，需要解包的有效载荷 |
| `CRC` | 校验字段，用于验证数据完整性 |
| `PADDING` | 填充字节，固定值，解析时跳过 |

### 数据类型（DATA 和 LENGTH 字段使用）

| 类型 | 字节数 | 说明 |
|------|--------|------|
| `uint8` | 1 | 无符号 8 位整数（0 ~ 255） |
| `int8` | 1 | 有符号 8 位整数（-128 ~ 127） |
| `uint16` | 2 | 无符号 16 位整数（0 ~ 65535） |
| `int16` | 2 | 有符号 16 位整数（-32768 ~ 32767） |
| `uint32` | 4 | 无符号 32 位整数（0 ~ 4294967295） |
| `int32` | 4 | 有符号 32 位整数 |
| `float` | 4 | IEEE 754 单精度浮点数 |
| `double` | 8 | IEEE 754 双精度浮点数 |

### CRC 算法

| 值 | 说明 |
|----|------|
| `CRC8` | CRC-8，多项式 0x07 |
| `CRC16_MODBUS` | CRC-16/Modbus，初始值 0xFFFF，多项式 0xA001 |
| `CRC16_CCITT` | CRC-16/CCITT，初始值 0xFFFF，多项式 0x1021 |
| `CRC32` | CRC-32，多项式 0xEDB88320 |
| `NONE` 或留空 | 不校验 |

### LENGTH 含义

| 值 | 说明 |
|----|------|
| `TOTAL` | 长度值 = 整帧总字节数（含帧头、帧尾、CRC 等所有字段） |
| `PAYLOAD` | 长度值 = 仅数据区字节数（不含帧头、帧尾等非 DATA 字段） |

> 如果帧大小固定（无 LENGTH 字段），解析器会自动按配置中所有字段的字节数之和计算帧大小。

### 大小端说明

| 值 | 含义 | 举例：数值 `0x1234` |
|----|------|---------------------|
| `BIG` | 高字节在前（大端序） | 内存中为 `12 34` |
| `LITTLE` | 低字节在前（小端序） | 内存中为 `34 12` |

> 大小端配置影响 DATA、LENGTH、CRC 字段的字节序解析。uint8 / int8 只有一个字节，大小端无影响。

## 四、配置示例

以下配置定义了一个 26 字节的帧：

```
帧头(2B) + 温度(1B) + 时间戳(4B) + 速度(4B) + 电压(2B) + 比率(8B) + 填充(1B) + CRC(2B) + 帧尾(2B)
```

**CSV 格式：**

```csv
序号,字段名称,字段类型,数据类型,字节数,固定值,大小端,CRC算法,CRC起始字段序号,CRC结束字段序号,LENGTH含义,备注
1,帧头,HEADER,-,2,55AA,-,-,-,-,-,固定帧头
2,温度,DATA,uint8,1,-,-,-,-,-,-,摄氏度
3,时间戳,DATA,uint32,4,-,BIG,-,-,-,-,毫秒时间戳
4,速度,DATA,float,4,-,LITTLE,-,-,-,-,m/s
5,电压,DATA,int16,2,-,LITTLE,-,-,-,-,mV有符号
6,比率,DATA,double,8,-,BIG,-,-,-,-,无量纲比率
7,填充,PADDING,-,1,00,-,-,-,-,-,填充字节
8,CRC,CRC,-,2,-,LITTLE,CRC16_MODBUS,2,6,-,校验字段2-6
9,帧尾,TAIL,-,2,0D0A,-,-,-,-,-,固定帧尾
```

**对应的帧数据（十六进制）：**

```
55 AA  1A  00 00 03 E8  C3 F5 48 40  DC 00  3F EB 33 33 33 33 33 33  00  9F 32  0D 0A
│───│  │─│ │─────────│  │─────────│  │───│  │───────────────────────│  ││  │───│  │───│
帧头   温度   时间戳       速度       电压         比率              填充  CRC   帧尾
       26    1000       3.14        220        0.85
```

## 五、TXT 数据文件格式

数据文件为纯文本，内容为十六进制字节值，支持以下格式（可混用）：

| 格式 | 示例 |
|------|------|
| 带 `0x` 前缀，空格分隔 | `0x55 0xAA 0x1A 0x00` |
| 不带前缀，空格分隔 | `55 AA 1A 00` |
| 逗号分隔 | `55,AA,1A,00` |
| 换行分隔 | 每行若干字节 |

文件中可以包含任意非帧数据（垃圾字节），解析器会自动搜索帧头进行定位。

---

## 六、串口设置

| 参数 | 可选值 | 默认值 |
|------|--------|--------|
| 波特率 | 9600 / 19200 / 38400 / 57600 / 115200 / 230400 / 460800 / 921600 | 115200 |
| 数据位 | 5 / 6 / 7 / 8 | 8 |
| 停止位 | 1 / 1.5 / 2 | 1 |
| 校验位 | None / Even / Odd / Space / Mark | None |

使用流程：
1. 点击「刷新」获取可用串口列表
2. 选择串口和参数
3. 点击「打开串口」开始接收数据
4. 状态栏会显示已缓存的字节数
5. 数据接收完毕后点击「开始解析」
6. 点击「关闭串口」停止接收

---

## 七、解析结果说明

解析结果表格的列：

| 列 | 说明 |
|----|------|
| 帧序号 | 从 1 开始的帧编号 |
| 帧偏移 | 该帧在原始数据流中的字节偏移量 |
| CRC | CRC 校验结果：`OK` 或 `FAIL`（无 CRC 字段时始终为 OK） |
| 数据字段... | 配置中所有 DATA 类型字段的解析值，列名为字段名称 |

---

## 八、导出格式

导出的 TXT 文件格式如下：

```
UART数据解析结果
总帧数: 5
========================================

--- 帧 #1 (偏移: 4, CRC: OK) ---
  帧头: 55 AA
  温度 = 26  [原始: 1A]
  时间戳 = 1000  [原始: 00 00 03 E8]
  速度 = 3.140000  [原始: C3 F5 48 40]
  电压 = 220  [原始: DC 00]
  比率 = 0.850000  [原始: 3F EB 33 33 33 33 33 33]
  填充: 00
  CRC: 9F 32
  帧尾: 0D 0A

--- 帧 #2 (偏移: 29, CRC: OK) ---
  ...
```

## 九、测试验证

项目 `test/` 目录下提供了两组配套测试文件，可通过 `python test/generate_test_data.py` 重新生成。

### 测试用例 1：固定长度帧（无 LENGTH 字段）

帧结构：`帧头(2) + 温度(1) + 时间戳(4) + 速度(4) + 电压(2) + 比率(8) + 填充(1) + CRC16(2) + 帧尾(2) = 26 字节`

| 文件 | 说明 |
|------|------|
| `test_config.xlsx` / `test_config.csv` | 配置文件 |
| `test_data.txt` | 测试数据，5 帧 + 垃圾数据 + 不完整帧 |

**预期结果：**

| 帧 | 温度 | 时间戳 | 速度 | 电压 | 比率 | 测试点 |
|----|------|--------|------|------|------|--------|
| 1 | 26 | 1000 | 3.14 | 220 | 0.85 | 正常值 |
| 2 | 30 | 2000 | 6.28 | -150 | 1.23 | 负 int16 |
| 3 | 18 | 3500 | 0.0 | 0 | 0.0 | 全零值 |
| 4 | 255 | 4294967295 | -1.5 | -32768 | 99.999 | uint32/int16 极值 |
| 5 | 0 | 0 | 100.0 | 32767 | -0.001 | uint8/int16 边界 |

CRC 算法：CRC-16/Modbus，校验范围为字段 2-6，所有帧 CRC 均为 OK。

数据中包含：帧前垃圾 `DE AD BE EF`、帧间垃圾 `AA BB CC`、末尾不完整帧 `55 AA 00`。

---

### 测试用例 2：带 LENGTH 字段的帧

帧结构：`帧头(2) + 帧长度(2) + 设备ID(1) + 角度(2) + 加速度X(4) + 加速度Y(4) + CRC8(1) + 帧尾(1) = 17 字节`

| 文件 | 说明 |
|------|------|
| `test_config_with_length.xlsx` / `test_config_with_length.csv` | 配置文件 |
| `test_data_with_length.txt` | 测试数据，5 帧 + 垃圾数据 + 不完整帧 |

**配置要点：**

```csv
序号,字段名称,字段类型,数据类型,字节数,固定值,大小端,CRC算法,CRC起始字段序号,CRC结束字段序号,LENGTH含义,备注
1,帧头,HEADER,-,2,EB90,-,-,-,-,-,固定帧头
2,帧长度,LENGTH,uint16,2,-,BIG,-,-,-,TOTAL,整帧总长度
3,设备ID,DATA,uint8,1,-,-,-,-,-,-,设备编号
4,角度,DATA,int16,2,-,BIG,-,-,-,-,0.01度有符号
5,加速度X,DATA,float,4,-,LITTLE,-,-,-,-,m/s^2
6,加速度Y,DATA,float,4,-,LITTLE,-,-,-,-,m/s^2
7,CRC,CRC,-,1,-,-,CRC8,3,6,-,校验字段3-6
8,帧尾,TAIL,-,1,FE,-,-,-,-,-,固定帧尾
```

- **帧长度字段**（序号 2）：类型为 `LENGTH`，数据类型 `uint16`，大端序，`LENGTH含义` 设为 `TOTAL` 表示该值 = 整帧总字节数（此例中 LENGTH 值固定为 17 = 0x0011）
- **CRC 算法**：CRC-8，校验范围为字段 3-6（设备ID + 角度 + 加速度X + 加速度Y）

**预期结果：**

| 帧 | 设备ID | 角度 | 加速度X | 加速度Y | 测试点 |
|----|--------|------|---------|---------|--------|
| 1 | 1 | 4500 | 9.81 | 0.0 | 正常值 |
| 2 | 2 | -9000 | -3.5 | 2.7 | 负角度 |
| 3 | 255 | 18000 | 0.0 | 0.0 | 180度，零加速度 |
| 4 | 0 | -18000 | -9.81 | -9.81 | -180度，自由落体 |
| 5 | 128 | 0 | 100.5 | -100.5 | 零角度，大加速度 |

CRC 算法：CRC-8，所有帧 CRC 均为 OK。

数据中包含：帧前垃圾 `FF FF`、帧间垃圾 `00 00`、末尾不完整帧 `EB 90 00 05`（帧头 + 错误长度值）。
